Design Notes

For this section, I want to show my design wireframes, flowcharts, intended experience, and the rest of that juicy design stuff. To start off with, I want to talk a bit about my design style process. Of course, that starts with my intended audience, which is a creative company that would be investigating me for portfolio purposes. To me, this means that there would be two important aspects to keep in mind. First, I want to make sure that my website is impressive technically. Of course it won't be the most incredible website ever created, but showcasing at least some competency in website design will be important for showing companies that I can create what I want to create, given enough time. Secondly, I think that user experience will be even more important than that, so I plan to center good semantic markup and clear, accessible content as much as I possibly can. This leads into my wireframes, and I think that it will be best to sort them by page, so I can show inspirations where necessary and demarkate them legibly. Additionally, I'm not sure what to put in terms of microformats and related things. As an aside, it seems like the software I was using for doing my flowcharts darkened them. I'm not sure what to do about that, so I apologise! However, before speaking about specific pages, I want to talk about my style guide.

Style Guide

The style guide was something I had thought of as endemic to wireframes, but I'm adding it here to do two things:

  • To show that it has indeed been considered.
  • To expand on parts that are not explored per-page, but rather site-wide.

I want to make this website reminiscent of a code editor. That means I want to have a color pallete and styling that feels almost like it's own language. To support this, I will be using Jetbrains mono as the primary font, as it is my favourite programming font, as well as being very well-known for legibility. Additionally, it is free and open-source. With that said, let's talk about colours!

I've never been very good at colours. I have a basic understanding of colour theory and I have spent a long time trying to improve, but even with all of that, I don't want to base a website on it, as while I'm confident I can put together something that makes sense, I think it will be a huge timesink. However, I do have a lot more experience with creating based on value. As such, my plan for this website is, essentially, shades of gray. Practically, this means that a dark gray will be used for the backgrounds of pages. White text will be the contrast color that is used, and I plan to manage values so that the proper elements are emphasised at the right time and place. I also want to create good pops of color. This will help break up the monotony of the site as well as help create a visual language that's intuitive and understandable.

The next important piece of styling is website feel. The primary guiding word that is being used here is "Clean." Sleek and intuitive navigation and input is key to achieving this feeling. Embedding content rather than redirecting the user will help to enforce flow. By making sure every page follows the same general theme, the user will not have their immersion interrupted.

Homepage

For my homepage, I want to create a strong statement piece. Something that says "Here I am! Look at me!" In this design, I am very inspired by this portfolio page that I found by the French graphic designer and creative director of Jam3, Steven Mengin. I'll also link it here. I really like his animated background which is incredibly visually interesting while staying entirely readable because of good color contrast. There is notably no part of the shifting texture that can ever reach a white as bright as the font onscreen, which means the name is always distinct. In terms of UX, I don't like how the website has animations for scrolling, as that takes away some agency from the user in terms of navigation. However, I do like how the entire portfolio is on one page, and it only takes two clicks at most to reach any interactive element of the site.

These are my wireframes for the homepage and the different aspect ratios that I want to make it for. Namely, I want to have it work for iPad, iPhone, and desktop. I want my landing to be similar to the inspiration image that I showed, however, I want to have some tricks with opacity, where my about page (the grey box and text) fades in as you get to it. The same concepts that I had originally, I plan to keep the same. For that reason, and to not repeat myself forever, that's where I'll leave my plans for this page. The flow diagram is incredibly simple, as the only interactible element is the interaction bar at the top. I expect a user to simply scroll to the bottom then click the bar to navigate to other pages from there.

Included here as well is my wireframe for how I plan for the menu to work. On click, it will open sideways to show each page tab as a clickable object. If one of the categories has subnavigation, such as for a specific blog or essay, they will display as a scrollable secondary dropdown.

Blog Page

For my blog section, I don't want there to be too much to distract the user. I want to center the reading experience and make interacting with blogs simple and intuitive, continuing on the same philosophy of the rest of the site. To facilitate this, there will not be a lot of visual activity to distract the reader beyond opening and closing of blog posts. Here, I want to mention an accessibility tradeoff that is being made for the sake of uniqueness, which is that the plan for the blog pages themselves will not be to have them simply in plain text. Rather, there will be some visual stylings in and around them. This may negatively impact readability, in which case the plans may be changed in the future.

There is no inspiration for this page beyond keeping the same theming as the homepage and the rest of the website. I will be keeping a close eye on the amount of whitespace that it contains, and more importantly, the reading experience that it'll create. Importantly, I want to figure out more easy navigation between blogs, and so will add subnavigation to the hamburger bar. If possible, I would also like to add native support for font size changes and possibly plain text versions as well for accessibility purposes.

For the essay page, I plan to replicate this behaviour, but rather than expanding as text, it will load an embedded pdf, and I also pln to include a download option for that pdf. For the mobile and iPad aspect ratios, the summary will not appear.

Portfolio Page

For my portfolio, I want to have splash art on the full screen with contact information in the bottom corner, with the position being slightly adjusted for mobile to be in a more noticeable area. Then on user scroll, with the same feeling as my main page, I want to pull the user into the actual portfolio section, tabbed for the two sections that are planned. Again, I want the mobile version of this page to have the tabs in a different area so that they can be switched between more easily. Additionally, if possible, I want to set up those tabs to remain on the top or bottom of the screen when the user scrolls down, so that they can switch views whenever they want. This will probably necessitate a deviation form the wireframe a little bit, so that the navigation bar doesn't interfere with the tabs. Hopefully, I can integrate 3D viewers and include textures to a custom element, but that's subject to change in the event that there isn't enough time. I don't expect that to be an issue though.

Final Mockups

For this section, I will be showing off the Mockups for the final looks of this website, as well as comparing them to the original wireframes. To keep accessibility in mind, I will be using this color-blindness simulator to make sure that visual elements can be seen clearly by anyone.

Starting off with the menu, the final design is smooth and minimalist. The currently selected tab has a color of ##66c3cc. This is the same color as certain objects in jetbrains rider, in keeping with the coding theme. Every other colour is derived from the pallette. By darkening the background of the selected tab as well as changing it's colour, this is readable even in grayscale and any of the tested colorblind modes.

This screen keeps with the code theme in that I'm using the colors from the rider editor. #39cc8f for the green (class name), #6c95eb for the dark blue (parameter declaration) and #66c3cc for the light blue (parameters). Once again, this screen is readable in all colorblind versions, including grayscale. The values of the coloured headings are darker than the lightest colour in the pallette, but they still are bright enough to contrast nicely with the dark colour that was chosen here. I would like to make a typing animation for a lot of these letters if possible. Additionally, I will keep the opacity idea from before.

This screen showcases the style of the website best. It has been filled with lorem ipsum text to show what a blog post might look like. I plan to override the inbuilt strong and em functionality for display so that they don't change the look of the text. Then I am once again using the rider code colours #39cc8f for the green (function), #66c3cc for the light blue (parameters), #6c95eb for the dark blue (declaration), and #c191ff for the purple (class name). This, to me, makes the page feel clean and sleek, as well as keeping in tune with the code theme. If time allows, I want to animate the underscore after Blogs: to blink like an insert cursor in the terminal. Additionally, I would like to attempt to create typing animations on opening blogs and loading the buttons.

This page is slightly less powerful in terms of keeping with the code theme. This is because images aren't normally displayed in code. Rather, I have gone back to the style guide as much as I can. All colors are from the pallette, and the purple and blue are the same jetbrains colours as before. I have kept in borders around the image to show that even if a piece is of a nonstandard size or shape, they will be auto-fitted to the grid. Additionally, I might add title text to the bottom of each piece that links to the corresponding instagram post.
